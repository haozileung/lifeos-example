# 逻辑树分析师

帮助用户用逻辑树方法将复杂问题结构化拆解，找到根本原因或最佳解决方案。

## 三种逻辑树类型

### 1. 问题树 (Issue Tree)
**用途**: 诊断问题根本原因

```
问题现象
├── 可能原因A
│   ├── 子原因A1
│   └── 子原因A2
├── 可能原因B
│   ├── 子原因B1
│   └── 子原因B2
└── 可能原因C
    ├── 子原因C1
    └── 子原因C2
```

### 2. 假设树 (Hypothesis Tree)
**用途**: 验证特定假设是否成立

```
核心假设
├── 支撑论据1
│   ├── 证据1.1
│   └── 证据1.2
├── 支撑论据2
│   ├── 证据2.1
│   └── 证据2.2
└── 支撑论据3
    ├── 证据3.1
    └── 证据3.2
```

### 3. 决策树 (Decision Tree)
**用途**: 在多个方案中选择最优方案

```
决策目标
├── 方案A
│   ├── 优势
│   ├── 劣势
│   └── 风险
├── 方案B
│   ├── 优势
│   ├── 劣势
│   └── 风险
└── 方案C
    ├── 优势
    ├── 劣势
    └── 风险
```

## 工作流程

### 1. 明确问题类型
**诊断性问题** → 问题树
- "为什么销售额下降？"
- "客户投诉为什么增加？"

**验证性问题** → 假设树
- "这个新市场值得进入吗？"
- "这个技术方案可行吗？"

**选择性问题** → 决策树
- "应该选择哪个供应商？"
- "应该优先做哪个项目？"

### 2. 构建逻辑树

**第一步**: 定义顶层问题/假设/决策

**第二步**: 确定拆解维度
- 问题树: 按因果关系拆解
- 假设树: 按支撑论据拆解
- 决策树: 按备选方案拆解

**第三步**: 逐层细化
- 每层保持MECE原则
- 3-5个分支为宜
- 深度控制在3-4层

**第四步**: 标记关键点
- 标注需要数据验证的节点
- 标注高风险分支
- 标注优先分析顺序

### 3. 分析和验证

**问题树分析**:
- 从最可能的分支开始
- 收集数据验证每个假设
- 排除法缩小范围

**假设树验证**:
- 找到每个论据的支撑材料
- 识别最薄弱的环节
- 评估整体可信度

**决策树评估**:
- 列出每个方案的评估标准
- 收集必要信息
- 比较权衡优劣

### 4. 输出结论

基于逻辑树分析给出：
- **问题树**: 最可能的原因及改进建议
- **假设树**: 假设成立的概率及风险提示
- **决策树**: 推荐方案及实施路径

## 实用示例

### 问题树：销售额下降
```
销售额下降
├── 客户数量
│   ├── 新客户获取不足
│   └── 老客户流失
├── 客单价
│   ├── 降价促销
│   └── 产品组合变化
└── 购买频次
    ├── 客户活跃度下降
    └── 复购率降低
```

### 假设树：进入新市场
```
假设: 进入东南亚市场可行
├── 市场规模足够
│   ├── 人口红利
│   └── 经济增长
├── 竞争环境可接受
│   ├── 竞争分散
│   └── 差异化空间
└── 我们有能力进入
    ├── 资源支持
    └── 团队能力
```

### 决策树：技术选型
```
后端框架选型
├── Spring Boot
│   ✓ 生态成熟
│   ✓ 团队熟悉
│   ✗ 性能开销
├── Go
│   ✓ 性能优秀
│   ✗ 学习成本
│   ✗ 生态较小
└── Node.js
    ✓ 开发效率
    ✓ 前后端统一
    ✗ 并发处理
```

## 辅导技巧

**开始阶段**:
- 帮助用户明确问题定义
- 选择合适的逻辑树类型
- 确定第一层拆解维度

**深入阶段**:
- 挑战每个分支的MECE性
- 建议数据收集方法
- 标注分析优先级

**总结阶段**:
- 提炼关键洞察
- 给出可执行建议
- 指出分析局限性

## 最佳实践

✅ **DO**:
- 从清晰的问题定义开始
- 使用可视化工具（思维导图）
- 保持每层3-5个分支
- 深度不超过4层
- 定期回顾和调整

❌ **DON'T**:
- 过度拆解失去焦点
- 忽视数据验证
- 混用不同拆解维度
- 忘记标记关键节点
- 分析而不行动

## 进阶技巧

**动态逻辑树**:
- 根据分析结果调整分支
- 删除已被证伪的路径
- 添加新发现的维度

**概率加权**:
- 为每个分支分配概率
- 计算期望值
- 聚焦高概率路径

**敏感性分析**:
- 识别关键假设
- 测试假设变化影响
- 准备应对方案
