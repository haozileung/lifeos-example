# Resources

> A resource is a topic or theme of ongoing interest.

## 📂 资源文件夹结构

```
3.Resources/
├── PKM/                      # 元知识：个人知识管理系统
├── Enterprise-Architecture/  # 组织与战略：EA、数字化转型、本体建模、组织治理
├── SoftwareArchitecture/     # 技术工程：架构模式、DDD、系统设计
├── Cloud-DevOps/             # 技术工程：云原生、DevOps、网络工具
├── Data-Management/          # 技术工程：数据治理、数据中台
├── Security/                 # 技术工程：安全工程、OWASP
├── AI/                       # 智能系统：Agent、Skills、LLM
├── Thinking-Methods/         # 个人成长：思维方法、认知科学
├── Professional-Growth/      # 个人成长：技术领导力、职业发展、沟通
├── Coach/                    # 个人成长：教练技术、转化模型
├── Philosophy/               # 个人成长：生活禅、人生哲学
└── Productivity/             # 效率工具：GTD、OKR、产品规划
```

## 🏷️ 标签体系

| 前缀 | 域 | 说明 |
|------|---|------|
| `resources/` | PARA 分类 | 资源文件夹主标签 |
| `ea/` | 企业架构 | TOGAF、SOA、数字化转型、本体建模 |
| `sa/` | 软件架构 | 架构模式、DDD、建模、低代码 |
| `infra/` | 基础设施 | 云原生、DevOps、Serverless、网络 |
| `data/` | 数据 | 治理、中台、主数据、资产 |
| `security/` | 安全 | 认证、OWASP、密钥 |
| `ai/` | 人工智能 | Agent、Skills、自动化 |
| `pkm/` | 知识管理 | PARA、CETDE、模板 |
| `growth/` | 个人成长 | 思维、领导力、职业、教练、哲学 |
| `productivity/` | 生产力 | GTD、OKR、沟通、产品规划 |

## 📖 核心概念

| 概念 | 定义 | 关联域 |
|------|------|--------|
| **资源** | 有持续兴趣的主题，不一定有截止日期或完成标准 | 全域 |
| **主题** | 按知识领域而非行动性组织的基本单元 | 各子域 |
| **知识图谱** | 通过类型化关系链接构建的语义化知识网络 | 全域 |
